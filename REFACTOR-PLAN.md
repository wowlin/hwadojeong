# 리팩토링 계획 — 화도정 3D 모델 코드

작성일: 2026-07-15 · 갱신: 2026-07-15(3차 독자 재검사 + 4차 교차검증 병합·상충 판정 + 5차 독립 교차검증 병합 + 6차 이중 세션 잔여 차분 병합·층 정정) · 대상: `src/` 전체 + 검증 게이트 + 죽은 산출물 · 원칙: **화면(렌더 결과)은 1픽셀도 안 바뀐다**(예외: 표시 버그 수리 — §2-2 J·§5-11)

## 1. 목표 (요청 5가지)

1. 프로젝트 코드 전체 검사
2. 이후 코딩이 편리하도록 구조 개선
3. 너무 큰 파일 제거 — 편집·검색 시간 단축
4. 매직넘버 최소화 — 기준값에서 파생시켜, 기준 하나 고치면 관련 값이 전부 따라오게
5. 반복 부품(콘센트·실링팬·의자 등) 공통 모듈화 — 같은 부품은 한 벌만 정의하고 가져다 쓴다

### 2-2. 발견된 문제 (검사 결과 — 전 항목 코드 실물 재검증 완료 · 신규-3차 항목은 전수 통독 + grep 재검증)

**A. 거대 단일 파일** — 어떤 편집이든 3,909줄에서 검색해야 함. 편집 왕복 시간의 주범.

**B. "스코프 밖이라 값 고정" 하드코딩** — 한 파일인데도 블록 스코프에 갇혀, 이미 계산된 값을 숫자로 다시 박은 곳들. 코드 스스로 주석으로 자백하고 있음:
- 실외기실 벽 위치 `0.40`, `1.80` — 실제로는 「게스트룸2 뒤끝」「화장실 앞벽」인데 숫자 고정 (주석: "스코프 밖이라 값 고정")
- 홈리프트 안방쪽 면을 s2 외벽 코드가 `(외벽+계단폭)+9단+1.2+1.5`로 **재계산** — 계단 코드가 이미 계산한 값의 사본
- 복도창 X중앙 `리프트면 − 2.025`가 **5곳**에 복제 (1층·2층·3층 콘센트·창)
- `기초윗면+슬래브두께`(1층 바닥 윗면), `2·3층 바닥 레벨`, `천장 밑면` 재계산 5곳+
- 에어컨 위치: 설치 코드 `창뒤끝(+2.2)+이격 0.3` vs 콘센트 쪽 `+2.5` — **재검증 결과 두 값은 현재 동일**(2.2+0.3=2.5). "이미 어긋난 사본"은 오진. 다만 같은 값을 두 곳이 따로 계산하는 사본 구조라 창이 움직이면 그때 어긋남 → 일반 단일 출처화 대상(§4-#10).
- (신규-5차 · 6차 층 정정) 좌표 사본 추가: 화장실 低X 안쪽면을 계단 블록과 메모가 서로 다른 식으로 재계산 2벌 — 재계산 실물은 **2층 메모 getter**(3층 아님·6차 실물 확인, 3층은 계단 블록 안에서 같은 값을 공유해 무사), s2 1층 개구 좌표(폴딩 반폭·기둥폭·우측창 간격)를 콘센트 블록이 외벽 블록과 별도로 재정의, 2층 콘센트 좌표에 침대 길이·내력벽 두께·이불장 깊이가 리터럴로 재기입.
- (신규-5차) **치수 라벨 문자열 하드코딩**: '기초 0.5m'(2곳)·'4.0m'·'1.0m'·'경사 33°' 등 화면 라벨 텍스트가 상수/포맷 헬퍼를 안 쓰고 숫자를 박음 — 상수를 바꾸면 라벨이 거짓말을 한다(메모엔 ⑯이 강제하는 원칙이 라벨엔 미적용).
