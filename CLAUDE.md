# CLAUDE.md — 작업 규칙 (화도정)

이 규칙은 매 세션 강제 적용. 위반 = 사용자 시간 낭비.

## 0-0. 무조건 간결·명확 (모든 세션·프로젝트 전체 기본)
- **질문엔 핵심 답 한 줄.** 복창·배경·군더더기·자기변명·"참고로" 금지.
- 안 물어본 것 덧붙이지 마라. 제안은 꼭 필요할 때만 맨 뒤 1줄.
- 단정할 수 있으면 단정. caveat로 답 흐리지 마라.

## 0-1. 빠르게 — 왕복 최소화
- **느림의 원인은 검색·빌드·렌더의 왕복 횟수다.** 한 번에 하나씩 grep/read 하지 말고, 필요한 정보를 **한 번의 호출에 몰아서** 모은다(여러 grep을 한 명령에, 의존 없는 도구 호출은 한 응답에 병렬로).
- 코드 위치는 **추측 말고 한 번에 넓게 검색**해 좌표·상수·재질을 함께 확보한 뒤 곧장 수정한다.
- **빌드·렌더(`shot`)는 작업당 1회**가 원칙. 수정 전 여러 번 미리 돌리지 마라. 검증은 `npm test` → 필요한 화면만 1회 렌더.
- 같은 작업에 같은 검색을 반복하지 마라. 이미 읽은 값은 다시 안 읽는다.

## 0. 시킨 것만, 딱 그만큼
- 요청한 것 외에 **아무것도 바꾸지 마라.** "색만" = 색만(크기·두께·위치 그대로). "위치만" = 위치만.
- 추측으로 **추가 작업 금지.** 애매하면 1줄로 묻고 멈춰라. 추측해서 더 하지 마라.
- 사용자가 한 단계씩 지시하면 **한 번에 하나만** 처리. 묶지 마라.
- 시키지 않은 파일(특히 `src/main.js`) 건드리지 마라.
- 색상 변경 = 해당 `materials.xxx`의 **color 값 한 토큰만** 수정. 줄 재작성·그룹 이동·구조 변경 금지.

## 1. 검증 — 시각 확인은 사용자 몫
- `src/main.js` 편집 후 **`npm test`(문법·빌드)만** 돌린다. 통과 못 하면 보고 말고 고쳐라.
- **렌더/시각 확인은 절대 내가 하지 마라.** `npm run shot`·`shot:side`·캡처·이미지 열기 금지. 위치·정렬·기하가 맞는지는 **사용자가 직접 화면으로 확인**한다.
- 임시 캡처 스크립트(`_tmp-*.mjs` 등) **절대 만들지 마라.**
- 화면 결과를 내가 "맞다/됐다"고 단정하지 마라 — 시각 판단은 사용자에게 넘긴다.

## 2. 보고
- **모든 응답은 한글.** 코드·식별자·명령어 제외.
- 질문엔 **한 줄로 단정해서 먼저 답**하라. 요청 복창 금지. 번호별 요청은 번호별로 보고.
- **답을 단서(caveat)로 흐리지 마라.** 확신하면 "네/아니요"로 단정한다. 추가·제안은 **필요할 때만 답 뒤에 짧게 분리**하고, **추가가 답보다 길면 안 된다.** 안 물어본 것(아직 안 한 일·다른 제안)으로 본문을 채우지 마라.
- 안 된 걸 된 척 금지. 확인 못 한 건 "확인 못 했다"고 명시.
- **책임 회피 금지.** 이 세션에서 내가 만들거나 고친 코드를 "기존 코드"라고 떠넘기지 마라. 작성자가 의심되면 `git blame`·`git log -S`로 먼저 확인하고, 내 산출물이면 인정하고 고친다.
- **코드로 답하지 마라. 화면으로 답해라.** 사용자는 코드를 안 본다. 줄번호·변수명·파일경로·함수명을 답에 쓰지 마라(코드 확인은 내가 하고, 답변엔 안 쓴다). 답은 화면(렌더된 모델에 보이는 것)으로: 위치는 도면 방향(앞/뒤·좌우·거실쪽/안방쪽), 크기·치수는 m 단위로. 내부 구현이 원인이어도 사용자에겐 화면에 보이는 결과로 설명한다.

## 3. 자동 커밋
- `npm test` 통과 → **묻지 말고 즉시 커밋**. "커밋할까요?" 금지.
- 커밋 메시지: 한글 1줄 요약.

## 4. 하네스 (생성-검증)
- 검증은 **§1이 단일 출처.** 여기선 verifier 에이전트 호출하지 마라(중복). `npm test` 통과 → 커밋.
- 편집 전 체크리스트 (매 줄마다):
  1. 이 줄이 요청에 명시된 대상인가?
  2. 요청 문장에 이 변경을 뒷받침하는 단어가 있는가?
  3. 이 변경을 안 하면 요청 자체가 불가능한가?
  → 하나라도 No면 멈추고 묻는다.
- "화면: 요청" 형식 → 콜론 앞 = 대상 화면, 뒤 = 작업. 해당 화면에서 결과가 반드시 보여야 함.
- 에이전트: `.claude/agents/verifier.md` · 스킬: `.claude/skills/model-edit/SKILL.md` (둘 다 실재함).
- 종료 게이트 `scripts/stop-gate.sh`는 Stop 훅에 연결됨 — src가 dirty거나 기하 '완료' 단정 시 `npm test`·`check:render`·측면 렌더(최근성) 미충족이면 종료를 차단. 한글 보고 게이트(`korean-check.mjs`)도 함께 작동.
- 하네스 재구성·점검은 `harness` 플러그인 메타 스킬로: "하네스 구성해줘"/"하네스 점검해줘" 호출(에이전트 팀 모드, `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 설정됨).

## 5. 프로그램 구조 — 단일 출처·누적 (앞→뒤)
- **설계 파이프라인(배치→기초→바닥틀→바닥→계단→1층→다락→지붕)이 '진짜 집'이자 단일 출처다.** 각 부재는 처음 나오는 단계에서 **딱 한 번만** 정의한다. 뒤 단계는 그것을 자동 누적하고, 거기서 사용자가 할 일은 **'보이기/숨기기'뿐.** 같은 부재를 뒤 단계에서 다시 그리거나 좌표를 다시 잡지 마라.
- **참고 토글(+1층/+다락/+지붕/데크/썬룸/외벽/폴딩/악세사리/콘센트/측백담장/옆집담장)은 '부품 빠뜨림 방지 체크리스트'다. 위치용이 아니다.** 건드리지 말고, 필요한 부품이 있으면 거기서 **복사해** 파이프라인에 넣는다.
- **뒤 단계에서 앞 단계 산출이 안 따라오면 사용자 문제가 아니라 '단일 출처로 안 묶인 내 코드' 문제다.** 설계 작업의 일부로 조용히 단일 출처로 묶어라. "적용 안 됐으니 해달라"고 사용자가 다시 부르게 하지 마라. **사용자는 설계만, 구조 코드는 내가.**
- 새 설계는 **항상 단일 출처로 구현**해 자동 누적되게 한다. "됐다" 전에 **다락·지붕 단계까지 렌더로 확인**한다.
- 한 곳만 고치고 "끝"이라 하지 마라 — 같은 부류(형제) 중복을 **스스로 전수 점검**해 함께 정리한다.

## 5-1. 콘센트 = 각 층 '외벽' 그룹 (예외 없음)
- **앞으로 추가할 모든 층의 모든 콘센트(내부·외부 불문)는 반드시 그 층의 '외벽' 메뉴 그룹에 넣는다.** s2 1층→`s2Wall1Objects`(bF1Wall), 2층→`s2Wall2Objects`(bF2Wall), 3층→`s2Wall3Objects`(bF3Wall). 싱크대·가구 등 다른 토글에 넣지 마라.
- `captureInto(s2Wall<층>Objects, …)` 블록에 그린다. 좌표 재계산 시 이중 수집 금지(한 그룹에만).

## 5-2. 메모 숫자 = 코드 연동 (예외 없음)
- **메모(NOTES)에 적는 모든 치수·수치는 코드 변수·상수에서 계산해 `${...}`로 넣는다.** 숫자 리터럴에 단위(m·mm·㎡ 등)를 붙여 박아두지 마라 — 코드가 바뀌면 메모도 자동으로 바뀌어야 한다.
- 불가피한 규정값(법정 이격·건폐율 한도 등 모델 부재가 아닌 값)만 예외. 그때도 `test/smoke.test.js` ⑯의 `ALLOW`에 **근거 주석과 함께** 등록해야 통과한다.
- 강제: `npm test` ⑯ '메모 숫자 = 코드 연동'이 NOTES 블록을 스캔해 하드코딩 치수를 차단한다.

## 도면 방향
- 입구·데크 = 아래(앞). 평면 좌 = 안방(高 x). 평면 우 = 거실+주방(低 x).
