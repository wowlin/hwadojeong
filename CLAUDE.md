# CLAUDE.md — 작업 규칙 (화도정)

이 규칙은 매 세션 강제 적용. 위반 = 사용자 시간 낭비.

## 0. 시킨 것만, 딱 그만큼
- 요청한 것 외에 **아무것도 바꾸지 마라.** "색만" = 색만(크기·두께·위치 그대로). "위치만" = 위치만.
- 추측으로 **추가 작업 금지.** 애매하면 1줄로 묻고 멈춰라. 추측해서 더 하지 마라.
- 사용자가 한 단계씩 지시하면 **한 번에 하나만** 처리. 묶지 마라.
- 시키지 않은 파일(특히 `src/main.js`) 건드리지 마라.
- 색상 변경 = 해당 `materials.xxx`의 **color 값 한 토큰만** 수정. 줄 재작성·그룹 이동·구조 변경 금지.

## 1. 검증
- `src/main.js` 편집 후 **`npm test`** 통과 필수. 통과 못 하면 보고 말고 고쳐라.
- 위치·정렬·기하 변경 → `npm run shot`으로 렌더 확인. 머릿속 좌표로 단정 금지.
- **높이·단차·계단·튀어나옴 등 "위/아래·앞/뒤로 얼마나" 문제는 평면 `shot`으로 안 보인다 → 반드시 `npm run shot:side`(측면 시점)로 `shot-side.png`·`shot-side-corner.png`를 직접 열어 눈으로 확인.** 평면만 보고 "맞다/이미 됐다" 단정 금지.
- **"이미 맞다 / 코드상 같다 / 안 고쳐도 된다"는 결론도 검증 대상.** 사용자가 "맞춰줘"라 했는데 안 고쳐도 된다고 판단되면, 그 판단을 측면 렌더로 먼저 증명하고 화면을 근거로 보고하라. 화면 확인 없이 "이미 맞다"고 우기지 마라. (Stop 게이트가 측면 미확인 시 종료를 차단함.)

## 2. 보고
- **모든 응답은 한글.** 코드·식별자·명령어 제외.
- 질문엔 **먼저 답**하고 짧게. 요청 복창 금지. 번호별 요청은 번호별로 보고.
- 안 된 걸 된 척 금지. 확인 못 한 건 "확인 못 했다"고 명시.
- **책임 회피 금지.** 이 세션에서 내가 만들거나 고친 코드를 "기존 코드"라고 떠넘기지 마라. 작성자가 의심되면 `git blame`·`git log -S`로 먼저 확인하고, 내 산출물이면 인정하고 고친다.
- **코드로 답하지 마라. 화면으로 답해라.** 사용자는 코드를 안 본다. 줄번호·변수명·파일경로·함수명을 답에 쓰지 마라(코드 확인은 내가 하고, 답변엔 안 쓴다). 답은 화면(렌더된 모델에 보이는 것)으로: 위치는 도면 방향(앞/뒤·좌우·거실쪽/안방쪽), 크기·치수는 m 단위로. 내부 구현이 원인이어도 사용자에겐 화면에 보이는 결과로 설명한다.

## 3. 자동 커밋
- `npm test` 통과 → **묻지 말고 즉시 커밋**. "커밋할까요?" 금지.
- 커밋 메시지: 한글 1줄 요약.

## 4. 하네스 (생성-검증)
- src/main.js 편집 후 **verifier 에이전트 호출** → PASS일 때만 커밋.
- 편집 전 체크리스트 (매 줄마다):
  1. 이 줄이 요청에 명시된 대상인가?
  2. 요청 문장에 이 변경을 뒷받침하는 단어가 있는가?
  3. 이 변경을 안 하면 요청 자체가 불가능한가?
  → 하나라도 No면 멈추고 묻는다.
- "화면: 요청" 형식 → 콜론 앞 = 대상 화면, 뒤 = 작업. 해당 화면에서 결과가 반드시 보여야 함.
- 에이전트: `.claude/agents/verifier.md` · 스킬: `.claude/skills/model-edit/SKILL.md`

## 도면 방향
- 입구·데크 = 아래(앞). 평면 좌 = 가족방(高 x)·안방쪽. 평면 우 = 거실+주방(低 x).

## 명령
- `npm test` 검증 · `npm run dev` 개발 · `npm run build` 빌드 · `npm run shot` 렌더 캡처
