# CLAUDE.md — 작업 규칙 (화도정)

이 규칙은 매 세션 강제 적용. 위반 = 사용자 시간 낭비.

## 0. 시킨 것만, 딱 그만큼
- 요청한 것 외에 **아무것도 바꾸지 마라.** "색만" = 색만(크기·두께·위치 그대로). "위치만" = 위치만.
- 추측으로 **추가 작업 금지.** 애매하면 1줄로 묻고 멈춰라. 추측해서 더 하지 마라.
- 사용자가 한 단계씩 지시하면 **한 번에 하나만** 처리. 묶지 마라.
- 시키지 않은 파일(특히 `src/main.js`) 건드리지 마라.

### 0-0. 편집 전 체크리스트 (매 편집마다 강제)
편집할 줄을 정한 뒤, **코드를 치기 전에** 아래 3개를 자문한다. 하나라도 No면 멈추고 묻는다.
1. **이 줄이 요청에 명시된 대상인가?** (예: "데크 포세린" 요청 → `deckStairs`는 대상 아님 → No)
2. **요청 문장에 이 변경을 뒷받침하는 단어가 있는가?** (없으면 No)
3. **이 변경을 안 하면 요청한 것 자체가 불가능한가?** (불편하지만 가능하면 No → 묻고 멈춤)

### 0-1. 색상 변경은 color 값 한 곳만 (강제)
- "색 바꿔/색 맞춰" 요청 = 해당 `materials.xxx`의 **color 값 그 한 토큰만** 수정. 끝.
- **금지**: 줄 통째 재작성, 그룹 이동(예: 골조→데크), 위치·구조 변경, 그 외 어떤 곳도 손대기.
- 편집 후 **`git diff`** 떠서 **바뀐 줄이 color 값 한 곳뿐인지 확인**하고, 그 diff를 보고에 첨부. 색 외에 한 글자라도 바뀌었으면 **즉시 되돌리고** 다시.
- 색과 무관한 변경(그룹 이동·버그픽스 등)이 필요해 보여도, 색 작업에 묶지 말고 **먼저 1줄로 묻고 멈춰라.**

## 1. 위치·정렬·기하 변경은 렌더로 눈으로 확인 (절대 머릿속 좌표로 단정 금지)
- `src/main.js`의 말뚝/기둥/치수/정렬을 바꾸면, "맞다"고 **추론으로 단정하지 마라.**
- `npm run shot` 으로 배치도 PNG를 렌더해 **직접 보고** 확인한 뒤 보고.
- 정렬은 **X(좌우)와 Z(상하) 둘 다** 확인. 한 축만 보고 판단 금지.
- "캐시/새로고침 때문"이라는 핑계 금지. 의심되면 직접 렌더해 사실 확인.

## 2. 변경 후 항상 검증
- `src/main.js` 편집 후 **`npm test`**(문법·빌드·용어·레이아웃 감사) 통과 확인. 통과 못 하면 보고 말고 고쳐라.
- 기하/정렬을 새로 보장해야 하면 그 불변식을 `scripts/audit-layout.mjs`에 **단언으로 추가**해서 `npm test`가 강제하게 하라(권고는 약하다).

## 3. 보고 방식
- **모든 응답은 한글.** 예외 없음(코드·식별자·명령어 제외). 영어 문장으로 보고·설명 금지.
- 질문엔 **먼저 답**하고 짧게. 장황·핑계 금지.
- "끝났다" 말하기 전에 0·1·2 했는지 자문.

### 3-1. 안 된 걸 된 척 금지 (정직 보고)
- "통과/완료"는 **사용자가 본 그 결과로 확인했을 때만** 말한다. `npm test` 통과·렌더 떴음만으로 "됐다" 단정 금지 — 요청한 그것이 실제로 그렇게 됐는지 렌더·diff로 본 뒤 보고.
- **확인 못 한 건 "확인 못 했다"고** 명시. 부분만 됐으면 "여기까진 됨 / 이건 미확인"으로 나눠 보고.
- 매 편집 후 PostToolUse 훅이 `src/main.js` diff를 띄운다. **그 diff에 요청 외 변경이 섞였는지 매번 확인**하고, 있으면 되돌린 뒤 보고.

## 도면 방향 (ORIENTATION.md와 동일)
- 입구·데크 = 아래(앞). 평면 좌 = 가족방(高 x)·안방쪽. 평면 우 = 거실+주방(低 x).
- 사용자가 "좌/우" 하면 이 방향 기준. 카메라 각도로 재해석 금지.

## 4. 매 요청 완료 후 자동 커밋 (강제)
- `npm test` 통과하면 **즉시 커밋**. 다음 요청 전에 반드시 커밋된 상태여야 한다.
- 커밋 메시지: 한글, 작업 내용 1줄 요약.
- 커밋 안 하고 다음 작업 넘어가는 것 = 규칙 위반.

## 명령
- `npm test` 검증 · `npm run dev` 개발 · `npm run build` 빌드 · `npm run shot` 배치도 렌더 캡처(렌더 확인용)
